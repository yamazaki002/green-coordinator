# README — «Зелёный координатор»

## 1. Назначение

Проект реализует алгоритмы управления распределением энергии и их формальную верификацию.

Реализованы два режима:

* `control` — штатное управление
* `safe_control` — управление при недоверенных или повреждённых данных

Система обеспечивает выполнение следующих требований:

1. Бесперебойное питание больницы
2. Питание школы в рабочее время
3. Защита накопителя от глубокого разряда

---

## 2. Структура проекта

```bash
coordinator-2.py
test_coordinator-2.py
```

* `coordinator-2.py` — алгоритмы управления
* `test_coordinator-2.py` — автоматическая проверка корректности

## 3. Требования

* Python 3.9 или выше
* Дополнительные библиотеки не требуются


## 5. Ожидаемый результат

При запуске тестов выводится отчёт, содержащий:

* результаты проверок по всем целям безопасности
* итоговую сводку прохождения тестов

Завершение программы:

* код `0` — все проверки выполнены успешно
* код `1` — обнаружены ошибки

---

## 6. Использование

Пример вызова:

```python
from coordinator-2 import control

sensors = {
    "time": 12,
    "solar_avail": True,
    "wind_strength": 60.0,
    "battery_level": 70.0,
    "hospital_demand": 100.0,
    "school_demand": 60.0,
    "houses_demand": 70.0,
    "factory_demand": 80.0,
}

result = control(sensors)
```

Результат представляет собой словарь состояний ключей:

* `ges`
* `solar`
* `wind`
* `battery`
* `hospital`
* `school`
* `houses`
* `factory`
